# Simulation Catalog & Golden Questions (Vision Engine)

이 문서는 Axiom의 Vision 모듈 내 What-if 엔진과 Root Cause 엔진이 기준으로 삼는 **핵심 도메인 질문(Golden Questions) 라이브러리**를 정의합니다.

사용자(의사결정자)가 직접 시나리오 변수를 설정하기 전, 즉각적으로 시스템 성능을 진단하고 미래를 예측할 수 있게 돕는 '사전 정의된 템플릿 카탈로그'입니다.

## 1. Golden Question 템플릿의 구조 (The Anatomy)

각 시뮬레이션 카탈로그 항목은 다음의 표준 규격을 따릅니다:
1. **Context (상황 대상)**: 분석하고자 하는 대상 노드(자산, 조직, 프로세스 등).
2. **Variable (조작 변수)**: 변경할 KPI 수치, 제약조건, 발생 빈도.
3. **Impact Metric (관측 지표)**: 변수 조작 시 변화를 측정할 1~3개의 핵심 성과 결과물.

```json
{
  "catalog_id": "SIM-SC-01",
  "domain": "Supply Chain",
  "golden_question": "만약 [물류센터 A]의 [가동률]이 [10% 하락]한다면, [전체 당일 배송 달성률]과 [페널티 비용]에 어떤 영향이 있는가?",
  "context": "Warehouse: A",
  "variables": {"capacity_utilization": "-10%"},
  "impact_metrics": ["same_day_delivery_rate", "penalty_cost_total"]
}
```

## 2. 도메인별 시뮬레이션 카탈로그 (v1.0)

### 2.1 재무/리스크 (Finance & Risk)
- **FIN-01**: "만약 [환율/금리]가 [5% 상승]할 시, [다음 분기 순이익율]과 [부채 상환 여력]은 어떻게 방어되는가?"
- **FIN-02**: "만약 [상위 3개 거래처]의 [대금 지급이 15일 지연]될 때, [운영 현금 흐름(OCF)] 단절 시점은 언제인가?"

### 2.2 운영/물류 (Operations & Supply Chain)
- **OPS-01**: "만약 [명절 성수기 주문량]이 [평소 대비 200% 증가]한다면, 현재 [인력 교대조]로 감당 가능한 한계 시간은?"
- **OPS-02**: "만약 [특정 공급망의 부품 입고]가 [2주 지연]될 때, [최종 조립 라커다운]까지의 버퍼 타임은 며칠인가?"

### 2.3 고객 서비스 (Customer Experience)
- **CS-01**: "만약 [월간 장애 발생건수]가 [현재 20건에서 50건]으로 늘어날 때, [콜센터 대기 시간 평균]과 [이탈률]의 상관관계는?"
- **CS-02**: "만약 [특정 서비스 불만 CS 비율]을 [10% 감축] 정책을 시행할 때, [재구매율 KPI] 달성 가능성은?"

## 3. 카탈로그의 활용 및 Lifecycle (Flywheel 연계)

1. **학습(Generate)**
   - 초기 설정: 위 정의된 기본 카탈로그가 시스템 부트스트랩 시 Neo4j 지식그래프와 매핑 로드됨.
   - 새로운 Golden Question 발견: NLP 인터페이스(Canvas)에서 사용자가 반복적으로 묻는 특정 `조작 변수` 패턴이 3회 이상 발견되면, Agent가 이를 '신규 카탈로그 후보'로 추천.
2. **실행(Operate)**
   - 사용자가 대시보드 진입 시, 시스템은 해당 사용자의 RBAC(Role)에 맞는 카탈로그 3개를 추천 질의 UI 플로팅.
   - 클릭 한 번으로 백그라운드 What-if 비동기 작업(Task) 생성 및 결과 렌더링.
3. **진화(Learn)**
   - 매분기, 카탈로그 실행 빈도 및 사용자 '채택률'을 평가하여 쓰이지 않는 템플릿은 Deprecate.
   - 적중률(시뮬레이션 예측 vs 실제 운영 결과 비교)을 통해 템플릿의 `Variable` 및 `Impact Metric` 엣지 가중치 재조정.

## 4. Root Cause (RCA) 연계 시나리오
시뮬레이션 카탈로그는 What-if(미래 예측) 뿐 아니라, RCA(과거 원인) 템플릿으로도 역방향 활성화(Toggle)가 가능합니다.

- **미래 설정 (What-if)**: "A가 10% 증가하면 B는 어떻게 되는가?"
- **과거 분석 (RCA)**: "B가 10% 하락했는데, A가 원인일 확률은 몇 %인가?" (DoWhy 라이브러리의 Counterfactual 백도어 기준 사용)
